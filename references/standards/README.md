# 客户/企业标准登记说明

审核工作流第 4 步（覆盖度核查）依赖本目录下的标准矩阵文件。把客户标准或 DV 大纲整理成矩阵放进来，skill 即可核对"大纲有、报告无"的漏项。

## 如何登记一个新标准

1. 复制 `_template.md`，重命名为：`<客户或项目代号>-<标准简称>-<版本年>.md`
   示例：`x客户-电池包DV大纲-2025.md`、`OEM-A-VW80000-2021.md`
2. 填写标准信息与试验项目矩阵：
   - "试验项目"与"判定准则"两列是审核的核心依据，**必须填**
   - "试验条件要点"选填：填写后审核时会对照报告声明的试验条件（DL-A06）
   - "样品数量"用于核对样本量是否满足判定规则（DL-D04）
   - "报告对应章节"与"覆盖状态"两列留空，由审核时填写
3. 一份标准一个文件；标准换版时新建文件，不要覆盖旧版（旧报告可能仍引用旧版）。

## 审核时如何选择矩阵

- 用户提供报告时说明了客户/项目 → 选文件名匹配的矩阵
- 未说明 → 列出本目录所有可用矩阵，请用户指定；用户不指定则跳过覆盖度核查并在审核报告中注明

## 注意

- 本目录内容属于客户/企业标准，注意密级；登记前确认允许存放在本机并被 AI 工具读取。
- 矩阵只需要"条目级"信息（项目、条款、判定准则、样本量），不需要抄写标准全文。
- `_` 开头的文件是模板，不参与审核。
