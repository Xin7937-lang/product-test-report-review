# 审核自动化分工提示

用途：告诉你（审核 agent）每条清单项**该靠什么完成**，避免两类浪费——脚本已覆盖的条目逐字重查（浪费时间），或必须读原文的条目只看脚本线索（漏判）。

## 一、脚本已覆盖（直接采用 .workpaper.md 第一部分线索，甄别即可）

| 脚本检查 | 覆盖的清单条目 |
|---|---|
| CHECK-1 占位符/待办 | DC-F05、DL-E01（部分）、CD-G07 |
| CHECK-2 日期逻辑 | DC-B02、DC-D03（校准过期）、CD-S08、CD-S03（日期层面） |
| CHECK-3 编号一致性 | DC-C05、CD-S04、CD-G10 |
| CHECK-4 单位规范 | DL-B01（部分）、CD-J01 |
| CHECK-5 判定一致性 | DC-G02、DL-C01、CD-S01（汇总 vs 文字层面） |
| CHECK-6 标准年号 | DC-E02、CD-G01、CD-G02 |
| CHECK-7 空白单元格 | DL-E01、CD-G07 |
| CHECK-8 模板必备章节 | 模板档案结构完整性（启用时） |
| --pair 跨报告对照 | DC-P05（判定不一致、样品编号交集） |

注意：脚本输出是**线索**，采纳前必须回第二部分原文核对上下文，防止误报写入报告。

## 二、半自动（脚本给线索，必须人工/语义确认）

- DC-D03 校准有效性：脚本只比对文本声明的日期，证书真伪、认可范围 → 人工核对项
- DC-C05/DL-C04 编号：脚本能发现写法不一致和疑似笔误，但哪个是正确编号 → 人工确认
- DL-B01 单位：脚本只查 kWh/℃/%/毫欧 几种，其它单位靠你读原文时留意

## 三、纯语义（必须读原文判断，脚本帮不上）

- **DL-A02~A06**：判定方向、临界值（**DL-A03 需自行计算 |实测−限值|/|限值| 是否 <5%**）、最严值判定、数据与结论矛盾的单项层面、试验条件对照
- **DL-C02/C03/C05**：文字 vs 表格 vs 曲线趋势矛盾、试验条件前后一致、同参数多处数值一致
- **DL-D01~D04**：结论是否超出数据支持、外推依据、失效闭环、样本量
- **DC-A/B 签署资质**、**DC-E01/E03/E04 追溯**、**DC-F 五要素与图表要素**、**DC-G 结论表述**、**DC-P01~P04 PPT 呈现**

## 四、易漏提醒（历史审核中 AI 最常漏的）

1. **DL-A03 临界值**：看到"实测值接近限值且判合格"就要算裕度，不要滑过去
2. **判定方向用反**："不大于/≥/≤"与数值比对逐条过，尤其限值单位与实测单位不一致时（如 MΩ vs Ω/V）
3. **结论章 vs 单项章**：结论写"无异常/全部通过"时，回单项逐条对一遍
4. **百分比基准**：容量保持率、SOC 未说明相对基准（DL-B04）
